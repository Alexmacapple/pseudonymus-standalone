    Installation

    # Cloner le depot
git clone git@github.com:Alexmacapple/pseudonymus-standalone.git
cd pseudonymus-standalone

# Installer (cree le venv + toutes les dependances + lance les tests)
bash install.sh

# Lancer le serveur
.venv/bin/python3 serveur.py

    Le coeur (JSON, CSV, TSV, TXT, MD) fonctionne sans aucune dependance. Les formats bureautiques (XLSX, ODS, DOCX, PDF) et le streaming sont installes automatiquement par le script.

    Formats supportés

    Toutes les dépendances sont installées automatiquement par bash install.sh.

    Mode ligne de commande (CLI)

    L'outil s'utilise directement dans le terminal, sans interface graphique.

    # Pseudonymisation (réversible, génère les correspondances)
python3 pseudonymise.py fichier.json --mapping mapping.json --pseudo

# Anonymisation (irréversible, pas de correspondances)
python3 pseudonymise.py fichier.json --mapping mapping.json --anon

# Mode fort + NLP (détection maximale)
python3 pseudonymise.py fichier.json --mapping mapping.json --fort --nlp --pseudo

# Avec détection technique (IP, JWT, MAC)
python3 pseudonymise.py fichier.json --mapping mapping.json --tech --pseudo

# Traiter tous les fichiers JSON d'un dossier
python3 pseudonymise.py dummy --mapping mapping.json --input-dir /chemin/dossier/ --pseudo

# Streaming pour fichiers volumineux (> 2 Go)
python3 pseudonymise.py gros-fichier.json --mapping mapping.json --chunk-size 5000 --pseudo

# Scoring RGPD (évaluation sans pseudonymiser)
python3 pseudonymise.py fichier.json --mapping mapping.json --score-only

# Test sur les 100 premiers enregistrements
python3 pseudonymise.py fichier.json --mapping mapping.json --dry-run

# Génération automatique de mapping
python3 pseudonymise.py fichier.json --mapping-generate

# Dépseudonymisation (restauration)
python3 depseudonymise.py fichier_PSEUDO.json --correspondances confidentiel/correspondances.csv

    python3 serveur.py --port 8090

    Le serveur démarre sur http://127.0.0.1:8090. Ouvrez cette adresse dans votre navigateur pour accéder à l'interface web décrite ci-dessous.

    Interface web

    Pour ceux qui ne sont pas à l'aise avec le terminal, l'interface web offre les mêmes fonctionnalités dans le navigateur.

    Premiers pas

    Votre premier test en 30 secondes :

    1. Allez dans l'onglet Pseudonymisation
    2. Collez ce texte dans le champ « Texte source » :
      Bonjour Marie Dupont, votre email est marie.dupont@example.fr et votre numéro 06 12 34 56 78.
    3. Cliquez sur Pseudonymiser
    4. Le résultat apparaît à droite :
      Bonjour [PERSONNE_1], votre email est [EMAIL_1] et votre numéro [TEL_1].

    Les données personnelles ont été remplacées par des jetons. La table de correspondances (onglet Correspondances) vous montre que [PERSONNE_1] = Marie Dupont, [EMAIL_1] = marie.dupont@example.fr, etc.

    Pour traiter un fichier complet

    1. Allez dans l'onglet Import fichier
    2. Sélectionnez votre fichier (upload) ou saisissez son chemin sur le disque (local)
    3. Cliquez sur Proposition de mapping automatique : le moteur inspecte votre fichier et propose un mapping automatique
    4. Vérifiez le mapping dans le textarea : les champs détectés comme sensibles sont-ils les bons ? Ajustez si besoin
    5. Cliquez sur Lancer le traitement
    6. Le fichier pseudonymisé est écrit sur le disque. Téléchargez le zip (fichier + correspondances)

    Pour restaurer un texte pseudonymisé

    1. Allez dans l'onglet Restauration
    2. Collez le texte contenant les jetons ([PERSONNE_1], [EMAIL_1]...)
    3. Chargez le fichier confidentiel/correspondances.csv généré lors de la pseudonymisation
    4. Cliquez sur Restaurer : les valeurs originales réapparaissent

    Guide page par page

    Pseudonymisation

    A quoi ça sert : coller un texte court et voir immédiatement le résultat pseudonymisé à côté. Idéal pour tester ou pour un texte isolé (email, courrier, paragraphe).

    Options de pseudonymisation
    Option Ce qu'elle fait Quand l'utiliser
    StandardDétecte les emails, téléphones, IBAN, noms avec contexte (« Bonjour Marie », « M. Dupont »), URLs, adresses, organisations, villesPar défaut, suffit dans la majorité des cas
    FortStandard + prénoms isolés (« Marie » seul), patronymes en majuscules (« DUPONT »), codes postaux, dates de naissance, plaques d'immatriculation, GPSQuand le texte contient des noms sans contexte (listes, tableaux) ou des données indirectes
    TechDétecte les adresses IP (v4/v6), adresses MAC, tokens JWT, clés APIQuand le texte contient des données techniques (logs, configurations)
    NLPUtilise l'intelligence artificielle (spaCy) pour détecter des noms que les regex ne trouvent pas (+2 à 5 % de détection)Quand le mode Fort ne suffit pas. Nécessite l'installation de spaCy
    WhitelistMots séparés par des virgules qui ne seront jamais pseudonymisésQuand des noms d'entreprise sont confondus avec des patronymes (ORANGE, PETIT, MARTIN...)
    BlacklistMots séparés par des virgules qui seront toujours pseudonymisésQuand un nom est protégé par un contexte institutionnel mais doit être pseudonymisé quand même

    Import fichier

    A quoi ça sert : pseudonymiser un fichier entier (JSON, CSV, XLSX, DOCX, PDF, TXT, MD...). C'est la page principale pour le traitement en masse.

    Modes d'import
    Mode Fonctionnement Quand l'utiliser
    UploadLe fichier est envoyé au serveur via le navigateurFichiers jusqu'à 200 Mo environ
    Chemin localVous saisissez le chemin du fichier sur votre disque. Le serveur le lit directementFichiers volumineux (au-delà de 500 Mo, privilégier le CLI avec --chunk-size)

    Le mapping est obligatoire pour traiter un fichier. Trois façons de le fournir :

    1. Proposition de mapping automatique (recommandé pour la première fois) : cliquez sur le bouton, le moteur inspecte votre fichier et propose un mapping automatique dans le textarea. Vérifiez et ajustez avant de lancer.
    2. Charger un fichier mapping : si vous avez déjà un fichier .json de mapping (fourni par un collègue, généré précédemment).
    3. Chemin du mapping (mode local uniquement) : saisissez le chemin du fichier mapping sur le disque.
    Options de traitement
    Option Ce qu'elle fait
    PseudonymisationRemplace par des jetons réversibles + génère les correspondances et un zip téléchargeable
    AnonymisationRemplace par des jetons irréversibles. Pas de correspondances, pas de retour possible
    Mode fortActive le pipeline complet (prénoms isolés, patronymes, codes postaux, etc.)
    TechDétecte aussi les données techniques (IP, JWT, MAC)
    NLPUtilise spaCy pour détecter des noms supplémentaires

    Scoring RGPD

    A quoi ça sert : évaluer le niveau de risque RGPD d'un texte avant de le pseudonymiser. Le scoring compte les données personnelles détectées et attribue un score.

    Modes de scoring
    Mode Ce qu'il analyse
    StandardEmails, téléphones, IBAN, noms avec contexte, URLs, adresses
    FortStandard + prénoms isolés, patronymes, codes postaux, dates de naissance
    Barème de scoring RGPD
    Catégorie Points par détection Exemples
    Direct3Noms, emails, téléphones
    Finance5CB, IBAN, NIR, numéro fiscal
    Tech2IP, JWT, MAC
    Indirect1Dates, villes, adresses

    Niveaux de risque : NUL (0), FAIBLE (<10), MODERE (<50), ELEVE (<100), CRITIQUE (100 et plus).

    Correspondances

    A quoi ça sert : visualiser la table de correspondances entre les jetons et les valeurs originales après une pseudonymisation. Permet de vérifier ce qui a été détecté.

    Fonctionnalités : recherche par jeton ou valeur, pagination, export CSV.

    Restauration

    A quoi ça sert : remettre les valeurs originales à la place des jetons (dépseudonymisation). Utile pour retrouver le texte original à partir d'un texte pseudonymisé.

    Options de restauration
    Option Ce qu'elle fait
    RestaurerUtilise les correspondances chargées depuis un fichier CSV
    Utiliser les correspondances en mémoireUtilise les correspondances de la dernière pseudonymisation effectuée dans cette session

    Le fichier CSV de correspondances est généré automatiquement lors de chaque pseudonymisation (dossier confidentiel/correspondances.csv).

    Glossaire

    Les termes utilisés dans l'outil, expliqués simplement.

    Bouton de l'onglet Import qui inspecte automatiquement votre fichier et propose un mapping. Le moteur lit les premiers enregistrements, détecte les champs qui ressemblent à des noms, emails, téléphones, etc., et pré-remplit le mapping. A vérifier avant de lancer le traitement.

    Remplacer les données personnelles par des codes irréversibles. Aucun retour possible : les correspondances ne sont pas générées.

    Mots à toujours pseudonymiser, même si le contexte les protège normalement. Exemple : « Victor Hugo » est protégé dans « école Victor Hugo », mais la blacklist force sa pseudonymisation.

    Un mot pseudonymisé par erreur. Exemple : « PETIT » (adjectif) détecté comme un patronyme. Les stopwords et la whitelist servent à les éliminer.

    Le texte de remplacement. Chaque valeur unique reçoit un numéro unique : [EMAIL_1], [PERSONNE_2], [TEL_3]. Si la même personne apparaît plusieurs fois, elle reçoit toujours le même jeton.

    Le mapping indique quels champs contiennent le prénom et le nom de la personne. Le moteur cherche ensuite ces noms dans les champs texte libre pour les pseudonymiser aussi.

    Fichier JSON qui décrit votre fichier : quels champs contiennent des données sensibles, lesquels contiennent du texte libre à analyser, quels mots protéger. Sans mapping, le moteur ne sait pas quoi pseudonymiser.

    Remplacer les données personnelles par des codes réversibles (ex : « Marie Dupont » devient [PERSONNE_1]). On peut revenir en arrière grâce à la table de correspondances.

    Règlement Général sur la Protection des Données. Le scoring RGPD évalue le risque d'un texte en comptant les données personnelles détectées et leur sensibilité.

    Mots courants que le moteur ignore pour éviter les faux positifs. « Bonjour », « Direction », « Pierre » (le matériau) ne sont pas pseudonymisés grâce aux stopwords.

    Certains champs JSON contiennent eux-mêmes du JSON sous forme de texte (JSON stringifié). L'unwrap ouvre ce contenu imbriqué pour l'analyser.

    Mots à ne jamais pseudonymiser, même s'ils ressemblent à des noms de famille. Exemple : ORANGE (l'entreprise, pas le patronyme), SFR, FREE.

    Questions fréquentes

    Non. Le serveur tourne sur 127.0.0.1 (votre machine uniquement). Aucune donnée n'est envoyée sur Internet. Tout le traitement est local.

    Pseudonymisation : les jetons sont réversibles. Un fichier de correspondances (confidentiel/correspondances.csv) permet de retrouver les valeurs originales.

    Anonymisation : les jetons ne sont pas réversibles. Aucune correspondance n'est générée. Impossible de revenir en arrière.

    En mode Standard, le moteur ne détecte que les noms avec un contexte : « Bonjour Marie », « M. Dupont », « De: Pierre Martin <email> ». Un prénom seul (« Marie ») ou un nom en majuscules sans contexte (« DUPONT ») nécessitent le mode Fort. Si le nom n'est ni dans les dictionnaires INSEE ni détecté par les regex, essayez le mode NLP.

    Dans l'onglet Import local, saisissez le chemin complet du fichier sur votre disque. Le serveur le lit directement sans passer par le navigateur. Un fichier de 118 Mo (31 891 enregistrements) se traite en environ 1 minute.

    Cliquez sur Proposition de mapping automatique dans l'onglet Import. Le moteur inspecte votre fichier et propose un mapping automatique. Le mapping apparaît dans le textarea : vérifiez que les champs détectés sont corrects, ajustez si besoin, puis lancez le traitement.

    Dans le champ Whitelist (onglet Pseudonymisation) ou dans le champ "whitelist" du mapping (onglet Import). Séparez les mots par des virgules. Exemple : ORANGE, SFR, FREE.

    Allez dans l'onglet Restauration. Collez le texte pseudonymisé, puis chargez le fichier CSV de correspondances (généré lors de la pseudonymisation dans le dossier confidentiel/). Cliquez sur Restaurer.

    Un indicateur numérique du risque de fuite de données personnelles dans un texte. Plus le score est élevé, plus le texte contient de données sensibles. Utilisez l'onglet Scoring RGPD pour évaluer un texte avant de le pseudonymiser.

    Certains mots courants sont aussi des patronymes INSEE : PETIT, BLANC, MARTIN, PIERRE. Si le mode Fort les pseudonymise, ajoutez-les dans la Whitelist (onglet Pseudonymisation) ou dans le champ "whitelist" du mapping. Le dry-run affiche les 20 mots les plus fréquemment matchés pour vous aider à calibrer.

    En mode chemin local, après le traitement, un zip est créé automatiquement. Il contient deux fichiers : le fichier pseudonymisé (fichier_PSEUDO.json) et la table de correspondances (confidentiel/correspondances.csv). Le bouton « Télécharger le zip » déclenche le téléchargement.

    Chaque type de donnée détectée reçoit un préfixe différent : [PERSONNE_1] pour les noms, [EMAIL_1] pour les emails, [TEL_1] pour les téléphones, [VOIE_1] pour les adresses, [VILLE_1] pour les villes, [ORGANISATION_1] pour les entreprises, [IBAN_1] pour les IBAN, [NIR_1] pour les numéros de sécurité sociale. La liste complète est dans la section Référence technique.

    Le moteur fonctionne sans installation pour JSON et CSV. Pour les autres formats (utilisables en ligne de commande), installez le paquet correspondant avec pip3 install. Par exemple : pip3 install openpyxl pour Excel, pip3 install python-docx pour DOCX, pip3 install pdfplumber pour PDF.

    Le mode Standard traite 31 000 enregistrements en ~1 minute. Le mode Fort est plus lent (vérifie chaque mot contre les dictionnaires INSEE). Conseils : commencez par le mode Standard, n'activez Fort que si nécessaire. Utilisez Chemin local au lieu d'Upload pour les gros fichiers. En ligne de commande, --chunk-size 5000 permet de traiter des fichiers de plus de 2 Go sans saturer la mémoire.
