    Documentation

    Outil de pseudonymisation 100% local. Aucune donnée personnelle ne quitte la machine.

    Premiers pas

    Votre premier test en 30 secondes :

    1. Allez dans l'onglet Pseudonymisation
    2. Collez ce texte dans le champ « Texte source » :
      Bonjour Marie Dupont, votre email est marie.dupont@gmail.com et votre numéro 06 12 34 56 78.
    3. Cliquez sur Pseudonymiser
    4. Le résultat apparaît à droite :
      Bonjour [PERSONNE_1], votre email est [EMAIL_1] et votre numéro [TEL_1].

    Les données personnelles ont été remplacées par des jetons. La table de correspondances (onglet Correspondances) vous montre que [PERSONNE_1] = Marie Dupont, [EMAIL_1] = marie.dupont@gmail.com, etc.

    Pour traiter un fichier complet

    1. Allez dans l'onglet Import fichier
    2. Choisissez Chemin local si votre fichier fait plus de 50 Mo, sinon Upload
    3. Sélectionnez ou saisissez le chemin de votre fichier (JSON, CSV, Excel, DOCX ou PDF)
    4. Cliquez sur Analyser la structure : le moteur inspecte votre fichier et propose un mapping automatique
    5. Vérifiez le mapping dans le textarea : les champs détectés comme sensibles sont-ils les bons ? Ajustez si besoin
    6. Cliquez sur Lancer le traitement
    7. Le fichier pseudonymisé est écrit sur le disque. Téléchargez le zip (fichier + correspondances)

    Pour restaurer un texte pseudonymisé

    1. Allez dans l'onglet Restauration
    2. Collez le texte contenant les jetons ([PERSONNE_1], [EMAIL_1]...)
    3. Chargez le fichier confidentiel/correspondances.csv généré lors de la pseudonymisation
    4. Cliquez sur Restaurer : les valeurs originales réapparaissent

    Glossaire

    Les termes utilisés dans l'outil, expliqués simplement.

    Terme Définition
    Pseudonymisation Remplacer les données personnelles par des codes réversibles (ex : « Marie Dupont » devient [PERSONNE_1]). On peut revenir en arrière grâce à la table de correspondances.
    Anonymisation Remplacer les données personnelles par des codes irréversibles. Aucun retour possible : les correspondances ne sont pas générées.
    Jeton Le texte de remplacement. Chaque valeur unique reçoit un numéro unique : [EMAIL_1], [PERSONNE_2], [TEL_3]. Si la même personne apparaît plusieurs fois, elle reçoit toujours le même jeton.
    Mapping Fichier JSON qui décrit votre fichier : quels champs contiennent des données sensibles, lesquels contiennent du texte libre à analyser, quels mots protéger. Sans mapping, le moteur ne sait pas quoi pseudonymiser.
    Whitelist Mots à ne jamais pseudonymiser, même s'ils ressemblent à des noms de famille. Exemple : ORANGE (l'entreprise, pas le patronyme), SFR, FREE.
    Blacklist Mots à toujours pseudonymiser, même si le contexte les protège normalement. Exemple : « Victor Hugo » est protégé dans « école Victor Hugo », mais la blacklist force sa pseudonymisation.
    Stopwords Mots courants que le moteur ignore pour éviter les faux positifs. « Bonjour », « Direction », « Pierre » (le matériau) ne sont pas pseudonymisés grâce aux stopwords.
    Lookup noms Le mapping indique quels champs contiennent le prénom et le nom de la personne. Le moteur cherche ensuite ces noms dans les champs texte libre pour les pseudonymiser aussi.
    RGPD Règlement Général sur la Protection des Données. Le scoring RGPD évalue le risque d'un texte en comptant les données personnelles détectées et leur sensibilité.
    Unwrap Certains champs JSON contiennent eux-mêmes du JSON sous forme de texte (JSON stringifié). L'unwrap ouvre ce contenu imbriqué pour l'analyser.
    Analyser la structure Bouton de l'onglet Import qui inspecte automatiquement votre fichier et propose un mapping. Le moteur lit les premiers enregistrements, détecte les champs qui ressemblent à des noms, emails, téléphones, etc., et pré-remplit le mapping. À vérifier avant de lancer le traitement.
    Faux positif Un mot pseudonymisé par erreur. Exemple : « PETIT » (adjectif) détecté comme un patronyme. Les stopwords et la whitelist servent à les éliminer.

    Guide page par page

    Pseudonymisation

    À quoi ça sert : coller un texte court et voir immédiatement le résultat pseudonymisé à côté. Idéal pour tester ou pour un texte isolé (email, courrier, paragraphe).

    Option Ce qu'elle fait Quand l'utiliser
    StandardDétecte les emails, téléphones, IBAN, noms avec contexte (« Bonjour Marie », « M. Dupont »), URLs, adresses, organisations, villesPar défaut, suffit dans la majorité des cas
    FortStandard + prénoms isolés (« Marie » seul), patronymes en majuscules (« DUPONT »), codes postaux, dates de naissance, plaques d'immatriculation, GPSQuand le texte contient des noms sans contexte (listes, tableaux) ou des données indirectes
    TechDétecte les adresses IP (v4/v6), adresses MAC, tokens JWT, clés APIQuand le texte contient des données techniques (logs, configurations)
    NLPUtilise l'intelligence artificielle (spaCy) pour détecter des noms que les regex ne trouvent pas (+2 à 5% de détection)Quand le mode Fort ne suffit pas. Nécessite l'installation de spaCy
    WhitelistMots séparés par des virgules qui ne seront jamais pseudonymisésQuand des noms d'entreprise sont confondus avec des patronymes (ORANGE, PETIT, MARTIN...)
    BlacklistMots séparés par des virgules qui seront toujours pseudonymisésQuand un nom est protégé par un contexte institutionnel mais doit être pseudonymisé quand même

    Le fichier CSV de correspondances est généré automatiquement lors de chaque pseudonymisation (dossier confidentiel/correspondances.csv).

    Le mapping

    Le mapping est obligatoire pour traiter un fichier. Trois façons de le fournir :

    1. Analyser la structure (recommandé pour la première fois) : cliquez sur le bouton, le moteur inspecte votre fichier et propose un mapping automatique dans le textarea. Vérifiez et ajustez avant de lancer.
    2. Charger un fichier mapping : si vous avez déjà un fichier .json de mapping (fourni par un collègue, généré précédemment).
    3. Chemin du mapping (mode local uniquement) : saisissez le chemin du fichier mapping sur le disque.

    Options de traitement

    Option Ce qu'elle fait
    PseudonymisationRemplace par des jetons réversibles + génère les correspondances et un zip téléchargeable
    AnonymisationRemplace par des jetons irréversibles. Pas de correspondances, pas de retour possible
    Mode fortActive le pipeline complet (prénoms isolés, patronymes, codes postaux, etc.)
    TechDétecte aussi les données techniques (IP, JWT, MAC)
    NLPUtilise spaCy pour détecter des noms supplémentaires

    Scoring RGPD

    À quoi ça sert : évaluer le niveau de risque RGPD d'un texte avant de le pseudonymiser. Le scoring compte les données personnelles détectées et attribue un score.

    Mode Ce qu'il analyse
    StandardEmails, téléphones, IBAN, noms avec contexte, URLs, adresses
    FortStandard + prénoms isolés, patronymes, codes postaux, dates de naissance
    Catégorie Points par détection Exemples
    Direct3Noms, emails, téléphones
    Finance5CB, IBAN, NIR, numéro fiscal
    Tech2IP, JWT, MAC
    Indirect1Dates, villes, adresses

    Niveaux de risque : NUL (0), FAIBLE (<10), MODÉRÉ (<50), ÉLEVÉ (<100), CRITIQUE (100 et plus).

    Questions fréquentes

    Mes données sortent-elles de ma machine ?

    Non. Le serveur tourne sur 127.0.0.1 (votre machine uniquement). Aucune donnée n'est envoyée sur Internet. Tout le traitement est local.

    Quelle différence entre pseudonymisation et anonymisation ?

    Pseudonymisation : les jetons sont réversibles. Un fichier de correspondances (confidentiel/correspondances.csv) permet de retrouver les valeurs originales. Anonymisation : les jetons ne sont pas réversibles. Aucune correspondance n'est générée. Impossible de revenir en arrière.

    Pourquoi certains noms ne sont pas détectés ?

    En mode Standard, le moteur ne détecte que les noms avec un contexte : « Bonjour Marie », « M. Dupont », « De: Pierre Martin <email> ». Un prénom seul (« Marie ») ou un nom en majuscules sans contexte (« DUPONT ») nécessitent le mode Fort. Si le nom n'est ni dans les dictionnaires INSEE ni détecté par les regex, essayez le mode NLP.

    Comment traiter un fichier de plus de 100 Mo ?

    Dans l'onglet Import fichier, choisissez Chemin local au lieu de Upload. Saisissez le chemin complet du fichier sur votre disque. Le serveur le lit directement sans passer par le navigateur. Un fichier de 118 Mo (31 891 enregistrements) se traite en environ 1 minute.

    Je n'ai pas de mapping, comment faire ?

    Cliquez sur Analyser la structure dans l'onglet Import. Le moteur inspecte votre fichier et propose un mapping automatique. Le mapping apparaît dans le textarea : vérifiez que les champs détectés sont corrects, ajustez si besoin, puis lancez le traitement.

    Comment ajouter des mots à ne jamais pseudonymiser ?

    Dans le champ Whitelist (onglet Pseudonymisation) ou dans le champ "whitelist" du mapping (onglet Import). Séparez les mots par des virgules. Exemple : ORANGE, SFR, FREE.

    Comment restaurer un texte pseudonymisé ?

    Allez dans l'onglet Restauration. Collez le texte pseudonymisé, puis chargez le fichier CSV de correspondances (généré lors de la pseudonymisation dans le dossier confidentiel/). Cliquez sur Restaurer.

    Qu'est-ce que le score RGPD ?

    Un indicateur numérique du risque de fuite de données personnelles dans un texte. Plus le score est élevé, plus le texte contient de données sensibles. Utilisez l'onglet Scoring RGPD pour évaluer un texte avant de le pseudonymiser.

    Un mot a été pseudonymisé par erreur (faux positif), que faire ?

    Certains mots courants sont aussi des patronymes INSEE : PETIT, BLANC, MARTIN, PIERRE. Si le mode Fort les pseudonymise, ajoutez-les dans la Whitelist (onglet Pseudonymisation) ou dans le champ "whitelist" du mapping. Le dry-run affiche les 20 mots les plus fréquemment matchés pour vous aider à calibrer.

    Que contient le zip téléchargeable ?

    En mode chemin local, après le traitement, un zip est créé automatiquement. Il contient deux fichiers : le fichier pseudonymisé (fichier_PSEUDO.json) et la table de correspondances (confidentiel/correspondances.csv). Le bouton « Télécharger le zip » déclenche le téléchargement.

    Que signifient les jetons [VOIE_1], [ORGANISATION_1], etc. ?

    Chaque type de donnée détectée reçoit un préfixe différent : [PERSONNE_1] pour les noms, [EMAIL_1] pour les emails, [TEL_1] pour les téléphones, [VOIE_1] pour les adresses, [VILLE_1] pour les villes, [ORGANISATION_1] pour les entreprises, [IBAN_1] pour les IBAN, [NIR_1] pour les numéros de sécurité sociale. La liste complète est dans la section Référence technique ci-dessous.

    Comment installer les dépendances pour Excel, DOCX ou PDF ?

    Le moteur fonctionne sans installation pour JSON et CSV. Pour les autres formats, installez le paquet correspondant avec pip3 install. Par exemple : pip3 install openpyxl pour Excel, pip3 install python-docx pour DOCX, pip3 install pdfplumber pour PDF. La liste complète est dans la section Référence technique.

    Le traitement est lent, que faire ?

    Le mode Standard traite 31 000 enregistrements en ~1 minute. Le mode Fort est plus lent (vérifie chaque mot contre les dictionnaires INSEE). Conseils : commencez par le mode Standard, n'activez Fort que si nécessaire. Utilisez Chemin local au lieu d'Upload pour les gros fichiers. En ligne de commande, --chunk-size 5000 permet de traiter des fichiers de plus de 2 Go sans saturer la mémoire.

    Structure du mapping

    Le mapping est un fichier JSON qui dit au moteur : « dans mon fichier, voici les champs qui contiennent des données personnelles ». Voici un exemple simple pour un fichier de clients :

    {
  "description": "Fichier clients",
  "champs_sensibles": {
    "nom": {"type": "nom", "jeton": "NOM"},
    "prenom": {"type": "prenom", "jeton": "PRENOM"},
    "email": {"type": "email", "jeton": "EMAIL"},
    "telephone": {"type": "tel", "jeton": "TEL"}
  },
  "texte_libre": ["commentaire"],
  "lookup_noms": {
    "prenom": "prenom",
    "nom": "nom"
  },
  "whitelist": ["ORANGE", "SFR"],
  "blacklist": []
}

    Avec ce mapping, un enregistrement comme :

    {"nom": "Dupont", "prenom": "Marie", "email": "marie@test.com",
 "telephone": "06 12 34 56 78", "commentaire": "Contacter Marie Dupont"}

    Devient :

    {"nom": "[NOM_1]", "prenom": "[PRENOM_1]", "email": "[EMAIL_1]",
 "telephone": "[TEL_1]", "commentaire": "Contacter [PERSONNE_1]"}

    Le champ commentaire est en texte_libre : le moteur y cherche automatiquement les noms, emails, telephones, etc. Le lookup_noms lui dit que le prenom et le nom de la personne sont dans les champs « prenom » et « nom », pour les retrouver aussi dans le texte libre.

    Champ du mapping Rôle Obligatoire
    champs_sensiblesListe des champs à remplacer directement par des jetons typésOui
    texte_libreChamps contenant du texte analysé par le moteur de détection (emails, noms, téléphones dans le texte)Non
    lookup_nomsIndique où trouver le prénom et le nom. Le moteur les cherche ensuite dans les champs texte libreNon
    whitelistMots à ne jamais pseudonymiser (noms d'entreprise : ORANGE, SFR...)Non
    blacklistMots à toujours pseudonymiser même si protégés par le contexteNon
    structure.unwrapPour les fichiers dont un champ contient du JSON imbriqué sous forme de texteNon

    Exemple réel : fichier SignalConso

    Ce fichier a une structure complexe : un champ RCLMFicheReportJsonSC contient du JSON sous forme de texte, avec des sous-champs comme Report.Firstname, Report.Email, etc. Le mapping utilise structure.unwrap pour ouvrir ce contenu :

    {
  "description": "Réclamations SignalConso",
  "structure": {
    "unwrap": {
      "field": "RCLMFicheReportJsonSC",
      "parse": "json_string"
    }
  },
  "champs_sensibles": {
    "DOAR_IDENT": {"type": "id", "jeton": "ID"},
    "Report.Firstname": {"type": "prenom", "jeton": "PRENOM"},
    "Report.Lastname": {"type": "nom", "jeton": "NOM"},
    "Report.Email": {"type": "email", "jeton": "EMAIL"},
    "Report.Phone": {"type": "tel", "jeton": "TEL"},
    "Report.PostalCode": {"type": "cp", "jeton": "CP"}
  },
  "texte_libre": ["Report.Description", "Report.Question"],
  "lookup_noms": {
    "prenom": "Report.Firstname",
    "nom": "Report.Lastname"
  },
  "whitelist": ["ORANGE", "SFR", "FREE", "BOUYGUES"],
  "blacklist": []
}

    Commandes CLI

    L'outil fonctionne aussi en ligne de commande, sans interface web.

    # Pseudonymisation (réversible, génère les correspondances)
python3 pseudonymise.py fichier.json --mapping mapping.json --pseudo

# Anonymisation (irréversible, pas de correspondances)
python3 pseudonymise.py fichier.json --mapping mapping.json --anon

# Mode fort + NLP (détection maximale)
python3 pseudonymise.py fichier.json --mapping mapping.json --fort --nlp --pseudo

# Avec détection technique (IP, JWT, MAC)
python3 pseudonymise.py fichier.json --mapping mapping.json --tech --pseudo

# Traiter tous les fichiers JSON d'un dossier
python3 pseudonymise.py dummy --mapping mapping.json --input-dir /chemin/dossier/ --pseudo

# Streaming pour fichiers volumineux (> 2 Go)
python3 pseudonymise.py gros-fichier.json --mapping mapping.json --chunk-size 5000 --pseudo

# Scoring RGPD (évaluation sans pseudonymiser)
python3 pseudonymise.py fichier.json --mapping mapping.json --score-only

# Test sur les 100 premiers enregistrements
python3 pseudonymise.py fichier.json --mapping mapping.json --dry-run

# Génération automatique de mapping
python3 pseudonymise.py fichier.json --mapping-generate

# Dépseudonymisation (restauration)
python3 depseudonymise.py fichier_PSEUDO.json --correspondances confidentiel/correspondances.csv

# Lancer le serveur web local
python3 serveur.py --port 8090
